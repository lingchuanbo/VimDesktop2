# 第二阶段结构升级说明

本次升级目标是先建立新骨架，再保留旧路径兼容，避免一次性大搬迁导致插件资源失效。

## 已完成

- 引入 `AppContext` 作为运行时上下文容器。
- 将插件发现统一到 `libs/PluginCatalog.ahk`。
- 为 `tools/` 与 `vendor/` 增加目录约定。
- 为现有插件增加 `runtime/*.entry.ahk` 入口包装层。
- `plugin.meta.ini` 统一改为指向 `runtime` 入口。
- `Blender` 的 `Script/`、`Send_to_Blender.py`、`external_runner_socket.py` 已迁移到 `plugins/Blender/assets/`。
- `AfterEffects` 的 `Script/` 与 `init_log.txt` 已迁移到 `plugins/AfterEffects/assets/`。
- `Max3D` 的 `Script/` 已迁移到 `plugins/Max3D/assets/`。

## 新约定

- 核心运行时代码:
  - `src/`
  - `libs/`
- 插件目录:
  - `plugins/<PluginName>/runtime/` 放入口包装和轻量运行时代码
  - `plugins/<PluginName>/assets/` 预留给资源迁移
  - `plugins/<PluginName>/` 现阶段继续保留旧资源和旧实现文件
- 工具与依赖:
  - `tools/` 放项目自管工具
  - `vendor/` 放第三方打包内容

## 兼容策略

- `PathResolver.AppsPath()` 现在优先查找 `tools/`，找不到再回退到 `apps/`。
- `PluginCatalog` 允许插件配置文件逐步迁移到 `runtime/`。
- 原插件主文件仍保留在原位置，`runtime` 入口只做转发，不改变行为。

## 后续建议

1. 逐个插件把路径拼接从 `A_ScriptDir ..\\plugins\\...` 改成 `PluginCatalog` / `PathResolver`。
2. 将可执行工具从 `apps/` 迁移到 `tools/`，每迁移一个就验证一个。
3. 将大型脚本资源逐步迁移到 `plugins/<PluginName>/assets/`。
4. 为 `AppContext` 增加统一的运行时读写接口，替代散落的全局变量直接访问。

## 当前保留项

- `EverythingWithOpen.ahk` 已迁移到 `plugins/Everything/assets/`，旧位置保留兼容转发文件。
- `TTOTAL_CMD.ini`、`Blender.ini`、`AfterEffects.ini` 等配置文件仍保留在插件根目录。
